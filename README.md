# Habits Tracker Project

Бэкенд-часть веб-приложения (SPA) для трекинга полезных привычек, спроектированная на основе книги Джеймса Клира «Атомные привычки».  
Проект разработан на базе **Django REST Framework (DRF)** и содержит асинхронную систему фоновых напоминаний в **Telegram** с использованием **Celery**.

---

## Технологии
* **Язык разработки:** Python 3.12+
* **Фреймворк:** Django / Django REST Framework (DRF)
* **Менеджер зависимостей и окружения:** Poetry
* **База данных:** PostgreSQL (основная) / Redis (брокер сообщений Celery)
* **Асинхронные задачи:** Celery / Celery Beat (планировщик)
* **Авторизация:** JWT (JSON Web Token) через `djangorestframework-simplejwt`
* **Документация API:** Swagger / ReDoc через `drf-yasg`
* **Тестирование и статистика:** APITestCase / Coverage / **unittest.mock (Mock, Patch)
* **Инструменты ручного тестирования и отладки:** Postman

---

## Бизнес-логика и валидаторы (Правила Джеймса Клира)
Бэкенд автоматически контролирует чистоту данных при создании/изменении привычек (`HabitSerializer`):
1. **Исключение дублирования наград:** Нельзя одновременно заполнить текстовое вознаграждение (`reward`) и связанную приятную привычку (`related_habit`).
2. **Правило двух минут:** Время выполнения атомной привычки (`duration`) ограничено 120 секундами.
3. **Чистота связей:** В качестве связанной привычки (`related_habit`) можно указывать только приятную привычку (с флагом `is_pleasant=True`).
4. **Приятная привычка самодостаточна:** У приятной привычки не может быть своего вознаграждения или своей связанной привычки.
5. **Регулярность выполнения:** Привычку нельзя выполнять реже, чем 1 раз в 7 дней (`periodicity <= 7`).

## Инструкция по локальному развертыванию

### Клонирование репозитория и подготовка окружения

1. Клонируйте репозиторий:
```
git clone https://github.com/AlexanderKokorinV/Habits_Tracker_Project.git
```
2. Установите зависимости:
```
poetry config virtualenvs.in-project true --local
poetry install
```

3. Конфигурация переменных окружения (.env)
Создайте в корне проекта файл **`.env`** на основе шаблона `.env.example` и заполните своими данными:

```
SECRET_KEY=my_SECRET_KEY #Ваш_SECRET_KEY
DEBUG=True

# Конфигурация базы данных
DB_NAME=my_data_base # Имя базы данных
DB_USER=username # Имя пользователя (например, postgres)
DB_PASSWORD=password # Пароль базы данных
DB_HOST=localhost # Хост для базы данных
DB_PORT=5432 # Номер порта, который закреплен за PostgreSQL по умолчанию

# Настройки для Redis
REDIS_URL=redis://127.0.0.1:6379/0

# Настройки Celery и Telegram
CELERY_BROKER_URL=redis://127.0.0.1:6379/0
CELERY_RESULT_BACKEND=redis://127.0.0.1:6379/0

TELEGRAM_URL=https://api.telegram.org/bot # Базовая ссылка для работы с телеграм-ботом
TELEGRAM_BOT_TOKEN=your_telegram_bot_token_here # Ваш токен бота, полученный от @BotFather

# Настройка разрешенных адресов (Origins)
CORS_ALLOWED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000,https://read-only.example.com,https://read-and-write.example.com
CSRF_TRUSTED_ORIGINS=http://localhost:3000,http://127.0.0.1:3000,https://read-and-write.example.com
```

4. Подготовка базы данных и миграции:

Перед запуском проекта убедитесь, что на вашем компьютере установлена и запущена СУБД **PostgreSQL**.
Создайте пустую базу данных, например, online_learning.

Выполните миграции Django для создания таблиц:
```
poetry run python manage.py makemigrations
poetry run python manage.py migrate
```

5. Создание суперпользователя (Администратора)

Создайте аккаунт администратора для доступа в панель управления. В качестве логина система запросит **Email**:
```
poetry run python manage.py createsuperuser
```

6. Запуск сервера разработки
```
poetry run python manage.py runserver
```

## Порядок запуска процессов проекта
Для полноценной проверки работы бэкенда и рассылки уведомлений необходимо запустить **3 параллельных процесса** в разных окнах терминала (убедитесь, что локальная служба Redis запущена на порту `6379`).

### Окно 1: Сервер веб-API (Django)
Принимает запросы от Postman/Фронтенда и обслуживает административную панель.
```
poetry run python manage.py runserver
```

### Окно 2: Очередь Celery Worker
Слушает брокер сообщений и выполняет непосредственную отправку сетевых HTTP-запросов к Telegram API.
```
poetry run celery -A config worker --loglevel=info
```

### Окно 3: Планировщик Celery Beat
Генерирует ежеминутные тики (импульсы) и ставит периодическую задачу рассылки в очередь воркера.
```
poetry run celery -A config beat --loglevel=info
```

## Архитектура эндпоинтов

### Пользователи и Аутентификация (`users`)
* `POST /users/register/` — Регистрация (возвращает данные профиля и JWT-токены мгновенно).
* `POST /users/token/` — Логин (получение пары `access` и `refresh` JWT-токенов по Email и Password).
* `POST /users/token/refresh/` — Обновление `access`-токена.
* `GET/PUT/PATCH /users/profile/` — Управление личным профилем (привязка `telegram_chat_id` для получения уведомлений).

### Трекер Привычек (`habits`)
* `GET /habits/` — Список личных привычек пользователя (Пагинация: по 5 записей на страницу).
* `POST /habits/` — Создание атомной привычки (прогоняется через 5 валидаторов).
* `GET /habits/public/` — Список публичных привычек других пользователей системы.
* `GET/PUT/PATCH/DELETE /habits/{id}/` — CRUD-управление конкретной привычкой (доступно только владельцу).
* `POST /habits/{id}/check/` — Отметка о выполнении привычки для SPA-интерфейса (генерирует запись в `HabitLog`).
